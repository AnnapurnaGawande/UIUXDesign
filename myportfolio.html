Project Overview

As a design student, I wanted to tackle a real problem I face weekly‚Äîgrocery shopping stress. This project became my opportunity to practice the full UX design process on a problem I genuinely cared about solving.

"Every Sunday, I'd spend 2 hours grocery shopping for what should be a 30-minute task. I'd forget items, wander aisles looking for products, and always overspend. I knew I wasn't alone in this struggle."

The Problem: My Grocery Shopping Reality

My Design Process

1

Week 1: Understanding the Problem

Research Methods:

  • Created simple survey using Google Forms
  • Conducted informal interviews with friends/family
  • Spent 3 hours observing in grocery stores
  • Sketched user journey maps

Biggest Learning: People don't need another complicated app‚Äîthey need something that actually makes shopping faster and less stressful.

2

Week 2: Ideation & Concepts

Brainstorming Process:

  • Used sticky notes to generate 50+ ideas
  • Grouped similar concepts
  • Voted on top 3 concepts with friends
  • Created simple user flows on paper

Selected Concept: A simple, smart shopping list app that organizes items by store layout, tracks spending in real-time, and works offline.

3

Week 3: Wireframing & Design

Lo-Fi Wireframes: Started with pen and paper sketches because it's faster for exploring ideas.

Design Decisions:

  • Large, thumb-friendly buttons (mobile-first)
  • Minimal color palette (green = go, red = stop)
  • Clear typography for easy scanning while shopping
  • Simple navigation with bottom tabs
4

Week 4: Prototyping & Testing

Prototype Creation: Built interactive prototype in Figma focused on core user flow: Create list ‚Üí Shop ‚Üí Complete

Testing Process:

  • Asked 6 friends to test the prototype
  • Observed them using it during actual shopping trips
  • Collected feedback via informal interviews
  • Iterated based on feedback

Testing & Iteration

User Testing Results

"Finally, a simple grocery app that doesn't try to do everything"
"The store section organization saved me so much time"
"Love how it works without internet‚Äîso many apps fail in stores"

Testing Method

Moderated usability testing with 6 participants

Task: Create a list, use it while shopping, track budget

Iterations Made

Round 1 Changes

  • Improved voice input with better error handling
  • Added subtle budget warnings instead of pop-ups
  • Simplified onboarding to 3 key screens
  • Made text larger for better readability

Round 2 Changes

  • Added quick tutorial animations
  • Improved item suggestions algorithm
  • Made checkout flow more prominent
  • Added dark mode option

What I Learned

Design Lessons

  • Start with the problem, not the solution
  • Simple is harder than complex
  • Test early and often
  • Design for real contexts

Process Lessons

  • Constraints breed creativity
  • User feedback is gold
  • Iteration is essential
  • Documentation helps thinking

Future Improvements

Next Steps

What I'd Do Differently
